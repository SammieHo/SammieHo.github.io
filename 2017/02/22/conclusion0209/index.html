正式提问

然后面试官说，问我一些问题~

HTML与CSS部分

  1. HTML5 的新特性，标签，接口等了解多少？列举一下

    • 首先是新的Doctype:<!DOCTYPE html>,浏览器按照标准模式去渲染页面，标准模式即是，对css解、HTML的排版，Js的运行模式皆以最高标准运行。
    • 新标签有：header,nav,aside,section,article,figure,figcapture,audio,video,mark等。
    • 新特性：
    1. 去掉link里的type属性
    2. 更多的表单特性，例如input里的email，pattern，placeholder
    3. 重新定义small，现用来表示下的排版，例如网站底部的版权声明

  2. 对websocket理解
    我说不知道0.0 他告诉我需要后端做数据的推送，前端负责接收数据，响应到页面。

  3. localstorage、sessionstorage与cookie的用法，区别，联系等都说说
    我从生命周期、存储大小、服务器端通信、应用场景来讲。
    原话还原：
    • 我先结合localstorage和sessionstorage来讲，首先他们都是HTML5关于存储的新特性。然后localstorage 是永久保存在客户端的，除非手动删除，而sessionstorage 在页面关闭时被清除，他们的存储大小都约5m，都是存在于客户端，不能与服务器端发生通信。
    • 而cookies 是携带在html的头部，可以设置时限，如果不设置，即在页面关闭后清除，存储空间是最小的，大约4k，具体根据浏览器而适当改变，可以与服务器端发生通信。
    • 最后，从应用场景来说，cookie主要运用在表单，登录状态等，而ls与ss的出现，分担了他的工作量。例如购物车，现在可以用ls来存储，而长表单就可以用ss来存储，分步骤的引导用户填写。而cookies专著与登录状态的判断。
  4. css的权值了解有多少？
    • 行类 1000
    • ID选择器 100
    • 类，伪类，属性选择器 10
    • 标签，伪类选择器 1
    • 通用选择器 ,子选择器> ,相邻选择器+ 为 0
      注意：important并没有权值！！！！
      注意：important并没有权值！！！！
      *注意：important并没有权值！！！！
  5. 描述从一个URL到页面呈现经历了什么，主要是前端这一块，特别是页面渲染过程。
    开始以为我一点都没答出来，后来回想，有答道一点，但并不系统，用词也不具体，因此重新总结：
    • 从Browser接收到html->解析；
    • 代码从上到下开始构建DOM树；
    • 根据DOM树和CSS规则书形成渲染树；
    • 最后将渲染树绘制到页面上。
      其中，当遇到script或link时会停止Dom树的构建，而去解析js/css；因为js可以通过DOM Api去修改Dom书的结构。

  4. 构造函数了解有多少？说列一个构造函数给他看看…
    我说：
    任何一个函数，当用new来实例化时就是一个构造函数。
    通常构造函数都是大写字母开头。用来保存属性的。

  5. 声明构造函数时，有一些原型与原型链的用法，你对他的理解有多少？你对他的应用是怎样子的？
    这里答得不是很好，主要讲了原型链的实现原理，知识把书本上看到的复述一遍，欠缺实践，其实原型链最大的作用就是用于继承。
    • 原型：
      js中万物皆对象，当然函数也是对象，当定义一个函数时，将会创建其prototype属性，指向函数的原型对象。
      当实例化一个对象时，就会把该实例的内部指针[[prototype]]指向原型对象，这一指向的引用就是原型。
    • 原型链：
      一个原型对象指向一个实例，该实例的proto属性指向另一个实例，层层推进，直到null。
      其实说这么多就是为了实现继承。
  6. window.onload 与 document.ready的区别？
    window.onload 是DOM文档加载完和所有文件加载完之后执行一个函数。
    document.ready 在文档书加载完之后执行一个函数。
